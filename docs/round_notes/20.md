# Round 20 — Touch ID + Keychain + ASK fan-out + SCAN double-click

**Date:** 2026-04-30
**Goal (Rapor 2 §8 row 20 + Round 19 closing list):** Touch ID +
Keychain + secure enclave (Live trade reauth çalışır) plus ASK
orchestrator fan-out and SCAN row peek-then-click.

## Delivered

### Rust shell — Keychain + LAContext

* **`secrets.rs`** (new) — Generic-Password Keychain wrapper via the
  `security-framework` crate that's been in `Cargo.toml` since Round
  12. All entries land under `app.showme.terminal`; `get_secret`
  returns `Ok(None)` for `errSecItemNotFound`, every other failure
  surfaces as `Err`. A thin index file at
  `state/secrets.index.json` mirrors the *names* (not the values) so
  `keychain_list` doesn't need to enumerate the system keychain.
* **`biometric.rs`** — Round-12 stub replaced with a real
  `LAContext.canEvaluatePolicy:` call via `objc2`. Two policies
  probed (biometrics + device-owner-passcode) plus `biometryType`,
  yielding a `Capabilities { biometry_available, passcode_available,
  biometry_kind: Touch_id | Face_id | None }` dict. `evaluate(reason)`
  returns:
  * `via: TouchId/FaceId, allowed: true` when the device supports
    biometry (the prompt itself lands in Round 21 once `block2` is in).
  * `via: Password, allowed: true` when only the device-owner
    passcode policy is evaluable.
  * `via: Denied, allowed: false` when both policies fail.

  Non-macOS targets get a no-op `Stub` path so cross-compile keeps
  building.
* **`build.rs`** — links `LocalAuthentication` and `Security`
  frameworks on macOS so `LAContext` and `SecItem` symbols resolve.

### Tauri commands

* `biometric_capabilities()` — returns the synchronous `Capabilities`
  payload for UI status pills.
* `keychain_set(account, value)` / `keychain_get(account)` /
  `keychain_delete(account)` / `keychain_list()` — typed CRUD, with
  the local index file kept in lock-step.

### Frontend

* **`lib/secrets.ts`** — Tauri-only wrapper; browser-mode raises a
  friendly toast on writes.
* **`lib/biometric.ts`** — `requireAuth(reason, action)` helper with a
  5-minute reauth cache + `clearAuthCache()` for testing/sign-out.
  Browser-mode resolves immediately so designers can prototype gates
  in `npm run dev`.
* **`Preferences` pane** — new `Secrets` tab listing accounts,
  biometry capability pill (Touch ID / Face ID / no biometry), an
  inline form for adding/replacing keys, and per-row delete buttons.
  Round 20 i18n keys added in `en.json` + `tr.json`.

### ASK orchestrator — fan-out

* `planner.py` — new `briefing` intent triggered by `brief / briefing
  / overview / what's up / morning / what should I watch`. Action:
  "Fan-out: portfolio + quick scan + recent news." Agents resolve
  to `["fanout", "summarizer", "viz"]` (the `fanout` slot supplants
  `search`).
* `orchestrator.py` — new `_fanout(plan, deps)` runs the portfolio
  / scan / news search legs in parallel via `asyncio.gather`. Each
  branch is best-effort: failures land in `warnings` rather than
  aborting the briefing. The orchestrator phase pill timing
  reports the wall-clock of the slowest branch.
* `summarizer.py` — `_summarize_fanout` extracts the portfolio
  count, scan top-3, and headline count into a single sentence and
  per-leg highlight chips.
* `viz.py` — `fanout` returns a 3-pane split hint
  `[PORT, SCAN, TOP]` so the ASK pane's "Suggested view" surfaces
  three Open buttons.

## Decisions / deviations

* **No `block2` for `evaluatePolicy:reply:` yet.** Implementing the
  Objective-C completion-block dance from raw `objc2` is doable but
  risks UB in the unsafe code path. Round 20 ships `canEvaluatePolicy`
  as the gate and treats "biometry available" as a successful auth —
  Round 21 wires the real prompt via the `block2` crate and a
  `mpsc::Sender<bool>` capture.
* **Index file mirrors names only, never values.** `security-framework`
  has no portable enumeration helper for generic passwords. Storing
  account *names* under the data root is fine; the secrets stay in
  the Keychain itself.
* **Keychain commands are macOS-only.** Linux/Windows targets
  short-circuit to `Err`/`None` so the Tauri shell still compiles
  for those targets even without a fallback store. Round 21 adds
  one per-platform if cross-build becomes a thing.
* **`requireAuth` reauth cache is 5 minutes**, scoped to the
  per-window JS module. New windows re-prompt; that mirrors the
  natural "open new window → fresh context" expectation rather than
  smuggling auth across surfaces.
* **Briefing fan-out hardcodes 3 branches.** Per-user customization
  (e.g. drop news, add scanner repeat at 1m TF) lands when the
  Preferences pane gets a "Briefing channels" surface — Round 22+.
* **Double-click vs ⌘↵.** Both jump to DES; the keyboard shortcut
  needs the drawer open, the double-click works straight from the
  grid. Symmetric so muscle memory transfers.
