# Round 33 — BTCUSDT function sweep and native stability

**Date:** 2026-05-01
**Goal:** User reported `CN` / Company News did not work for `BTCUSDT` and
suspected most functions were broken. Run a systematic BTCUSDT sweep, remove
systemic sidecar failures, and reinstall the native app.

## Delivered

* Added a persistent single ShowMe function worker loop. Cached async clients,
  locks, and SQLite-backed stores are no longer reused across random request
  event loops.
* `BTCUSDT`, `ETH-USD`, `SOL/USDC`, etc. now infer `CRYPTO` asset class when
  the caller does not explicitly pass `asset_class`.
* `/api/function-index` now loads in a background thread with a cache, so cold
  function inventory imports do not block `/api/health`.
* `/api/fn/{code}` now JSON-sanitizes pandas/numpy payloads before FastAPI
  serialization. DataFrame-returning functions no longer crash with 500.
* Function-internal provider exceptions now return a normal function payload
  with `warnings` instead of breaking the pane with HTTP 500.
* Frontend generic function timeout now matches the backend envelope
  (50s UI vs. 45s backend) so news/screen functions do not show false 15s
  failures.
* `CN` now calls the real `CN` backend for Company News, not the `NI` topic
  fallback path.
* Crypto `CN` uses RSS crypto/news feeds first and returns BTC headlines
  without Finnhub/GDELT keys. RSS summaries are stripped to plain text in UI.
* Added `feedparser` to sidecar dependencies and PyInstaller hidden imports.

## BTCUSDT sweep

* Dev sidecar full read-only BTCUSDT sweep:
  * `ok`: 64
  * `ok-warn`: 17
  * `warn-empty`: 38
  * `empty`: 12
  * `skipped-risk`: 7 trade/order functions
  * `http-error`: 0
  * `error`: 0
* Native packaged smoke:
  * `/api/health` returns OK immediately after the sidecar port is available.
  * `CN?symbol=BTCUSDT&asset_class=CRYPTO` returns 5 RSS headlines, 0 warnings.
  * `BRIEF` returns a news payload inside the new UI timeout.
  * `ACCT` and `PORT` see mirrored portfolio state in the installed app.
  * `FA` returns a payload with expected `No CIK for BTCUSDT` warning.
  * `BGAS` returns a warning payload for missing `EIA_API_KEY`, not HTTP 500.
  * Heavy screen functions that were slow inside a full-package sweep
    (`MOST`, `SECF`, `SECT`, `WEI`) return successfully when checked
    individually in the installed app; `SRCH` returns its expected Phase 5
    warning.
* Trade/order category (`AIM`, `BBGT`, `EMSX`, `EXEC`, `FXGO`, `TCA`, `TSOX`)
  was not executed because those surfaces can represent brokerage actions.
