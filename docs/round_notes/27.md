# Round 27 — Multi-broker scaffold (BaseBroker ABC + Alpaca paper)

**Date:** 2026-05-01
**Goal (direktif §0):** Cross-vendor broker abstraksiyonu — ShowMe'ün
PaperBroker'ı tek implementasyon olmaktan çıksın, Alpaca paper account
ilk gerçek adapter olarak gelsin. Round 28+ IBKR / Binance / Coinbase
adapter'ları aynı `BaseBroker` ABC üzerine kurulsun.

## Delivered

### `src-py/showme/brokers/base.py`

* **Enums** — `OrderSide` (buy/sell), `OrderType` (market/limit/stop/
  stop_limit), `TimeInForce` (day/gtc/ioc/fok), `OrderStatus` (new/
  accepted/filled/partially_filled/cancelled/rejected/expired).
* **`Order`** — submit_at, filled_at, avg_fill_price, raw payload;
  `to_dict()` enum'lar string'e çevirir.
* **`Position`** — entry_price, current_price, unrealized_pnl,
  realized_pnl, raw.
* **`BaseBroker` ABC** — async API: `account / list_positions /
  list_orders / submit_order / cancel_order / close_position`.
  `coerce_side / coerce_type / coerce_tif` static helper'ları
  string-or-enum kabul ediyor.
* **`BrokerError` / `NotSupported`** — adapter ortak hata sınıfları.

### `src-py/showme/brokers/paper.py`

`PaperBroker` — in-memory deterministic adapter:

* Cash + equity float; market order anında fill; limit order
  ACCEPTED'da bekler, `simulate_fill(id, price)` test hook ile filllenir.
* Notes hint pattern: `notes="last:200"` deterministic test fiyatı.
* Position güncellemesi: aynı yöne ekleme → ortalama maliyet
  recompute; ters yöne kapama → realized_pnl hesabı; net 0 → pozisyon
  silinir.
* Slippage / fee / margin yok — sadece şekil paritesi.

### `src-py/showme/brokers/alpaca.py`

`AlpacaPaperBroker` — Alpaca v2 REST, paper-trading:

* Env: `ALPACA_PAPER_KEY`, `ALPACA_PAPER_SECRET`,
  `ALPACA_PAPER_BASE_URL` (default `https://paper-api.alpaca.markets`).
* `http_call: HttpCall | None` enjekte edilebilir — testler stub
  callable veriyor; production'da `httpx` lazy import.
* `_alpaca_status()` Alpaca string → `OrderStatus` map; pending_*
  ACCEPTED, canceled/cancelled CANCELLED.
* `_to_order / _to_position` payload normalize; short pozisyon →
  `OrderSide.SELL`, qty mutlak değer.
* `cancel_order` 404/422 → False; diğer hata yeniden raise.
* `close_position` → DELETE `/v2/positions/{symbol}`; opsiyonel
  qty body.

### `src-py/showme/brokers/factory.py`

* `register_broker(name, factory)` — idempotent.
* `list_brokers()` — sıralı isim listesi.
* `get_broker(name|env|"paper")` → broker instance; bilinmeyen ad →
  `KeyError`.
* Built-in registrations: paper (her zaman); alpaca-paper (import
  başarılıysa).

### Sidecar `server.py` route'ları

* `GET  /api/broker/info?name=...` — broker name + registered list +
  account snapshot.
* `GET  /api/broker/positions?name=...`.
* `GET  /api/broker/orders?status=open&limit=100&name=...`.
* `POST /api/broker/orders` — JSON body (`symbol, side, quantity,
  order_type, time_in_force, limit_price, stop_price, notes, broker`);
  `BrokerError` → 400, beklenmeyen hata → 502.
* `DELETE /api/broker/orders/{id}?name=...` — boolean ok.

## Decisions / deviations

* **PaperBroker realized P&L hesabı net=0'da yapılır.** Kısmi kapama
  ortalama maliyeti korur (FIFO yok); Round 28+ FIFO/LIFO option'ları
  flag ile eklenir.
* **`http_call` enjekte edilebilir.** Adapter testleri httpx olmadan
  geçer; production'da lazy import. Aynı pattern Round 26 LLM
  modülüyle uyumlu.
* **AlpacaPaperBroker `name = "alpaca-paper"`.** Gerçek (live) Alpaca
  adapter'ı Round 28+ "alpaca-live" olarak ayrı kayıt; paper/live
  endpoint URL'i farklı.
* **`/api/broker/*` route'ları write-side dahil.** Direktif §0 "ham
  ürün" hedefi: scaffolding değil, gerçek POST/DELETE. UI tarafı
  Round 28'de PORT/TRAN pane'ler bağlanacak.
* **`SHOWME_BROKER` env ile default override.** Trader paper'dan
  alpaca-paper'a geçmek istediğinde kod değişikliği gerekmez.
* **`gateLiveTrade()` (Round 21 dondurulmuş) bu round'da kullanılmadı.**
  Live broker bağlantısı Round 28+ "alpaca-live" adapter'ı geldiğinde
  tekrar ısıtılacak.
