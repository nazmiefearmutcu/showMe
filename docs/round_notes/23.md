# Round 23 — TOP / ECO / WATCH / ALRT panes + NSAlert confirm

**Date:** 2026-04-30
**Goal (direktif §0 madde 2):** Trader'ın günlük 4 yüzey ihtiyacını
direkt karşılayan native pane'ler + Round 21'in donduruları yerine
NSAlert-tarzı `confirmAction()` helper'ı.

## Delivered

### `lib/confirm.ts`

* `confirmAction({ title, body, primary, cancel, destructive })`
  — Tauri shell'de `tauri-plugin-dialog.ask(...)` (native NSAlert),
  browser-mode'da fixed-position modal portal-style overlay.
* Single resolve API (returns boolean) so callers don't think about
  the transport.
* Used by ALRT delete-button to gate destructive actions; replaces
  `confirm()` (forbidden by ui_standards §6).

### TOP pane

* `useFunction("TOP")` polling + 60 s auto-refresh.
* Each headline as a `Card density="compact"` with title (15 px,
  letter-spacing -0.01em), source pill, sentiment pill, summary
  (≤240 chars), symbol chips → DES jump, ts label (`12m ago` /
  `2h ago` / ISO date), source link.
* `normalizeArticles()` accepts arrays or `{items|articles|headlines|news}`
  envelopes — Round 14's ShowMe-payload variance handled.

### ECO pane

* `useFunction("ECO", {days})` with segmented `This week / This month`
  toggle.
* Sortable date / region / event / importance / forecast / actual /
  surprise columns; `importanceBadge()` maps `high/med/low` text and
  numeric 1-3 to pill tones.
* `surprise = actual - forecast` rendered via `ChangeText` so beats /
  misses pop with the design-system color contract.

### WATCH pane

* `lib/watchlist.ts` — preset filesystem (Round 16) preferred, local
  storage fallback. `addSymbol/removeSymbol/saveWatchlist/loadWatchlist`.
* Pane defaults to 5 seed symbols (AAPL/MSFT/GOOGL/BTCUSDT/ETHUSDT)
  when the user opens it for the first time.
* Polls `/api/fn/DES` every 30 s for each row, lifts `regularMarketPrice`
  and `previousClose` into `last`, `change_pct`. Fail-soft: per-row
  `err` indicator stays red without aborting the loop.
* Symbol button → focused leaf jumps to DES via `setFocusedTarget`
  + hash route.

### ALRT pane

* `lib/alerts.ts` — same preset/localStorage adapter pattern; rows
  carry `{symbol, field, direction, threshold, note, fired_count,
  last_fired_at, active}`.
* Add form covers all four directions (`above / below / cross_up /
  cross_down`) + price / change% / volume fields.
* Toggle / delete / **test fire** actions per row. Test fire dispatches
  `notify` Tauri command (Round 16 `notifications.rs`) so the user can
  validate OS notification permissions without waiting for a real trip.
* Delete is gated on `confirmAction({...destructive: true})`.
* When OS notify is unavailable (browser-mode) the toast host is the
  fallback channel.

## Decisions / deviations

* **Watchlist polling, not WS.** Round 27+ promotes the loop to a
  Binance-WS / yfinance-WS bridge per the directive's real-time push
  item. Until then, 30 s polling per row is the lowest-friction surface
  the trader can immediately use.
* **Watchlist seeds 5 symbols on first open.** "Empty pane" is a bad
  first impression; the seed includes 3 equities + 2 crypto so both
  asset classes light up immediately. The user can `✕` them out.
* **Local persistence for WATCH and ALRT, not sidecar SQLite.** Round
  17 `presets.rs` already gives us a per-name JSON store; reusing it
  keeps the surface light and avoids a new Python migration. Round 27
  may flip to SQLite when alarm-engine cron/fan-out lands.
* **Alerts fire = toast + native notify, no order action.** Round 23
  alerts are observability-only. When Round 27's broker engine lands,
  alerts can optionally trigger a paper order via gateLiveTrade
  (which is frozen but still wraps the manual confirm modal).
* **`confirmAction` browser fallback is portal-style, not React.**
  The util is invoked from non-React contexts (e.g. CLI wrappers
  Round 24+) so a vanilla DOM overlay sidesteps any provider plumbing.
