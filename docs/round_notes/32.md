# Round 32 — Function execution, generic payload UI, native app

**Date:** 2026-05-01
**Goal:** Live browser smoke showed functions getting stuck in an endless
loading-looking state. Fix the function surface so every ShowMe function can
execute through `/api/fn/{code}`, improve the Bloomberg-like terminal layout,
and ship a working native `.app` in Applications.

## Delivered

### Function execution

* `FunctionStub` now calls `/api/fn/{code}` on pane open instead of rendering
  permanent skeleton placeholders.
* All generic function panes expose symbol input, Run/Go controls, OK/error
  state, elapsed time, sources, warning count, and raw payload inspection.
* Requests have a frontend timeout so a slow or wedged endpoint returns a
  real error state instead of an endless loading screen.
* Equity/chart/derivative-style functions now seed a default symbol from
  recent symbols, falling back to `AAPL`; `BETA` opens and executes directly.
* Nested result maps such as `BETA.betas` / period metrics are flattened into
  compact grids instead of one unreadable JSON line.
* Portfolio/global functions such as `ACCT` no longer show or send a symbol
  input; they execute against the local portfolio state and expose an optional
  Params JSON box for non-symbol arguments.
* Generic table cells now summarize nested arrays/objects (`51 positions`,
  `CRYPTO: 591,100...`) so account payloads stay readable while the raw JSON
  remains available in the payload drawer.

### Native and layout polish

* `Pane` switched from a fixed 3-row CSS grid to flex layout; native panes
  with a `SymbolBar` no longer push body/footer rows into the wrong area.
* Workspace restore now reapplies the current hash route after loading the
  saved layout, so `#/fn/DES` and `#/symbol/AAPL/DES` win over stale persisted
  pane state.
* Sidebar search/count/native-pane markers were tightened, with a Bloomberg
  orange accent rail and denser scan behavior.
* `DataGrid` gained stronger terminal-style row contrast and borders.

### Dependencies and native install command

* Added `yfinance>=0.2.40` to `src-py/pyproject.toml`; DES/BETA and other
  ShowMe equity functions now have the same Yahoo adapter dependency that ShowMe
  expects.
* Packaged the Python sidecar with PyInstaller and bundled ShowMe `src`,
  `dashboard`, and `config` directories into the native release.
* Frozen sidecar boot now moves relative `runtime/` writes into
  `~/Library/Application Support/showMe`, avoiding PyInstaller read-only
  extraction paths.
* Frozen sidecar boot mirrors small ShowMe runtime state files from the legacy
  ShowMe checkout into `~/Library/Application Support/showMe/runtime`, so
  portfolio/account functions can see the existing positions inside the native
  app.
* `ACCT` now avoids external quote lookups for imported crypto positions and
  falls back to average cost for unavailable quotes, preventing the previous
  45s timeout on crypto-heavy portfolios.
* Added Tauri command `install_to_applications`, exposed through
  `Preferences > About`; inside the native shell it copies the running
  `.app` bundle into `/Applications` idempotently.
* Added a Tauri v2 main-window capability file and hardened the frontend
  sidecar port handshake so native startup does not miss the first
  `sidecar:port` event.
* Added a warmup retry loop for cold `/api/function-index` loads; the sidebar
  recovers to 138/138 after the bundled ShowMe factory finishes importing.
* Built the macOS Tauri `.app`, copied it to `/Applications/showMe.app`, and
  ad-hoc signed the installed bundle for local execution.
