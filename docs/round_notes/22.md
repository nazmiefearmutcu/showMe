## Delivered

### `showme.migration` service

`src-py/showme/migration.py` — pure-Python, no third-party deps:

* `import_state(state, db_path, *, source="showme", writable=False)` →
  `MigrationSummary` dataclass.
* SQLite schema with three tables (`positions`, `trades`, `migrations`)
  + indexes on symbol.
* Idempotent: `INSERT … ON CONFLICT(symbol, side, opened_at) DO UPDATE`
  for positions, `(trade_id, symbol)` for trades. Re-running grows the
  audit log only.
* `--writable` CLI flag flips every imported row's `mode` column from
  `read_only` to `writable`. Default is read-only; this is Faz B
  Round-22 mirror, not Faz C primary store.
* `_iso_from_epoch` handles both seconds and milliseconds, ignores
  duration-shaped ints (< 10⁸), and falls through gracefully on
  non-numeric input.

* `commands::run_migration(enginePath?, writable?)` shells out to
  `python3 -m showme.migration` with the right `--to` path
  (`~/Library/Application Support/showMe/data/portfolio.db`) and
  parses the trailing JSON summary back into a `serde_json::Value`.

  read-only / writable mode picker + `Run import` button + summary

## Decisions / deviations

* **Pure SQLite, no DuckDB.** Migration runs once per machine; SQLite
  is plenty for 51 rows + ~50 KB of trade history. DuckDB lives next
  door for OHLCV (Round 23+ when MOST/HP/WEI need it).
* **Schema includes `raw_json`.** ShowMe's position dicts have keys
  Round 22 doesn't model directly (`liquidation_price`, `is_break_even`,
  …). Stashing the original JSON keeps audit honesty without
  pre-committing to columns.
* **`mode` column instead of separate writable/readonly tables.**
  Faz C will let the user "promote" individual positions to writable
  when they're ready to manage them via showMe; one column makes that
  a single UPDATE.
* **CLI shells out, not direct sidecar import.** The sidecar is
  long-lived; spawning a fresh Python process for migration keeps the
  import side-effect-free even if the sidecar's ShowMe path-import is
  stale. Round 27 may collapse this if the sidecar grows a "trigger
  migration" endpoint.
* **No `confirm()` on Migration writable toggle.** The Round-23 NSAlert
  helper isn't shipped yet; the writable toggle just flips a flag.
  When the alert lands we'll wrap the toggle. `mode='writable'` rows
  still need explicit "promote" before any future writer touches them.
