# Round 16 — Tray + dock + notifications + filesystem persistence

**Date:** 2026-04-30
**Goal (Rapor 2 §8 row 16):** "Tray + Dock + Notifications + Global
Shortcuts | macOS native his."

## Delivered

### Real Cocoa bridges via Tauri 2 first-party APIs

Round 12 left `dock::set_badge` / `dock::request_attention` as log-only
stubs because the manual `objc2` route was crash-prone in our deps tree.
Tauri 2 ships those methods on `WebviewWindow` directly:

* `dock::set_badge(count)` → `window.set_badge_count(Some(n))` /
  `set_badge_count(None)` clears.
* `dock::request_attention(critical)` → `window.request_user_attention(
  Some(UserAttentionType::Critical | Informational))`.

Both helpers are now generic over `R: Runtime`, take `&AppHandle<R>`,
and are no-op on non-macOS targets via Tauri's own platform fork.

### Tray live ticker (`src-tauri/src/tray.rs`)

Round 12's static menu (open / scan / alerts / quit) gained three
disabled "informational" items refreshed every 5 s by a Tokio interval:

* `Bot · RUNNING|STOPPED · cycle <n>`
* `Portfolio · <n> pos · $<compact-mv>`
* `Alerts · <n> active · <m> today`

The interval is spawned once on `tray::install`; it polls the sidecar
on `http://127.0.0.1:<port>/api/sidecar/ticker` (port discovered via
`AppState.sidecar_port`) using `reqwest 0.12 (rustls-tls)`. When
`alerts.active` changes value, the dock badge auto-syncs through
`dock::set_badge`.

### Sidecar `/api/sidecar/ticker`

Compact JSON used by the tray:

```jsonc
{
  "ts": "2026-04-30T18:27:33Z",
  "warnings": [],
  "bot":       { "running": false, "cycle": null, "mode": null },
  "portfolio": { "n_positions": 0, "daily_pnl": null,
                 "market_value": 0 },
  "alerts":    { "active": 0, "fired_today": 0 }
}
```

Each nested fetch (bot_service.get_state, PortfolioState.import_legacy,
alert_engine.list_alerts) is best-effort — failures append a string to
`warnings` and the rest of the payload still ships.

### Notifications wrapper (`notifications.rs`)

* `notify(app, NotifyOptions { title, body, thread, severity })` —
  `tauri-plugin-notification` builder + `group(thread)` for OS
  notification grouping (UNMutableNotificationContent.threadIdentifier).
* `Severity::Critical | Warn` cascades into `dock::request_attention`
  for the bounce.
* Action buttons (`UNNotificationAction`) deferred behind a feature
  flag in `tauri-plugin-notification 2.x`; the API shape is locked
  here so a future round flips one boolean.

### Filesystem-backed presets (`presets.rs` + Tauri commands)

Round 15's localStorage backend now has a sibling that writes JSON to
`~/Library/Application Support/showMe/state/layout-presets/<name>.json`.
Tauri commands:

* `list_presets() -> Vec<PresetSummary>`
* `read_preset(name) -> serde_json::Value`
* `write_preset(name, content)`
* `delete_preset(name) -> bool`

Name validation rejects `/`, `\`, `:`, control chars and empty input.

Frontend `lib/presets.ts` now picks the backend at call-time via
`isInTauri()`. Browser-mode keeps writing to localStorage so designers
running `npm run dev` still see the dropdown work.

### Workspace persistence (`workspace-persist.ts`)

A debounced (400 ms) zustand subscriber serializes the live tree on
every mutation:

* Tauri shell: `state/workspace.json` via `save_workspace` /
  `load_workspace` Tauri commands.
* Browser dev: `localStorage["showme.workspace"]`.

`restoreWorkspace()` runs once on App boot before the autosave
subscription installs, so a refresh keeps the user's split.

## Decisions / deviations

* **No manual objc2 dance.** Tauri 2 already wraps NSDockTile + NSApp
  request-attention. Less code, no unsafe, future Tauri upgrades pull
  in any platform fixes for free.
* **Tray ticker = disabled menu items, not a custom view.** macOS
  doesn't let `NSStatusItem` host arbitrary SwiftUI/AppKit views
  cleanly through Tauri 2 yet — `setText` on a disabled item is the
  cheapest way to ship a live ticker today. When Tauri 2 exposes
  `set_view` for tray items we can graduate.
* **5 s poll, not push.** A push channel from sidecar → Rust would
  need a websocket on a private port; for ticker resolution (cycle
  number + alert count) 5 s is plenty and the cost is negligible.
* **`reqwest` rustls-tls only.** No system-tls path → no OpenSSL
  cross-compile pain on macOS-arm64 CI runners.
* **Workspace autosave debounce 400 ms.** Drag-resize handle fires
  ~60 events/second; without debounce we'd write the JSON file on
  every frame. 400 ms is well above the human eye's threshold for
  perceived lag and well below "did my split survive a kill -9?"
  expectations.
* **Workspace state is per-window.** Two windows opened in the same
  session share a single autosave file (last-writer-wins). When we
  add per-window state in Round 17, we'll key by window label.
