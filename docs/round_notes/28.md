# Round 28 — Tauri auto-update + GitHub release manifest

**Date:** 2026-05-01
**Goal (direktif §0):** App'in kendisi güncellenebilir hale gelsin —
Tauri updater plugin `active=true`, signed artifact pipeline, manifest
JSON GitHub Releases üzerinden yayınlanır, kullanıcı Settings → "Check
for updates" ile yeni sürümü tek tıkla indirir.

## Delivered

### Tauri shell

* **`Cargo.toml`** — `tauri-plugin-updater = "2"` eklendi.
* **`tauri.conf.json`** — `plugins.updater.active=true`, endpoint
  `https://github.com/showme-app/showme/releases/latest/download/latest.json`,
  `dialog=true`, `pubkey="PLACEHOLDER_REPLACE_WITH_TAURI_GENERATED_KEY"`,
  Windows `installMode: passive`.
* **`src/lib.rs`** — plugin registered (`tauri_plugin_updater::Builder::new().build()`).
* **`src/commands.rs`** — yeni iki Tauri command:
  * `check_for_updates(app)` → `UpdateInfo {available, current_version,
    latest_version, release_date, release_notes, error}`. Updater
    plugin'i `app.updater().check()`'i sarmalıyor; tüm hatalar `error`
    field'ine düşer (panic etmez).
  * `apply_update(app)` → `download_and_install(...)` + `app.restart()`.
    Kullanıcı opt-in, dialog otomatik çıkmaz; UI'dan manual click.
  * `invoke_handler!` array'ine eklendi.

### Frontend — `lib/updater.ts` + Preferences About

* `checkForUpdates() / applyUpdate()` Tauri-only wrapper'lar; browser
  modunda stub sürer.
* `Preferences > About` artık "Check for updates" butonu, sonuç pill'i
  (positive/muted/warn), release notes preformatted blok ve "Download
  & restart" butonu (sadece `available && !error`) gösteriyor.
* `AboutSection` ayrı komponent olarak ayrıştırıldı (önceki inline
  kart silindi).

### Release pipeline

* **`scripts/build_release_manifest.py`** — release directory layout
  (`release/<platform>/{showme.dmg,showme.dmg.sig}`) → Tauri-uyumlu
  `latest.json` (version, notes, pub_date ISO-UTC, platforms map).
  Imza dosyası bulunmazsa platform atlanır + uyarı stderr'a düşer.
  CLI `--release-dir / --version / --notes-file / --upload-base / --out`.
* **`.github/workflows/release.yml`** — yeni workflow; tag push (`v*`)
  veya manual dispatch ile tetikleniyor:
  1. macOS-14 matrix (aarch64 + x86_64) Tauri build (signed via
     `TAURI_SIGNING_PRIVATE_KEY`).
  2. Bundles `release/<platform>/` altına stage edilir.
  3. Ubuntu publish job artifact'ları indirir, manifest'i üretir,
     `softprops/action-gh-release` ile dmg + .sig + latest.json
     GitHub Release'e yüklenir.
* `build-mac.yml` (Round 17'den) tag-push tetiğini paylaşıyor — yeni
  release.yml end-to-end signed pipeline.

## Decisions / deviations

* **`pubkey` placeholder** — `tauri signer generate` çıktısı CI
  secret olarak `TAURI_SIGNING_PRIVATE_KEY` (+ password) ile gizleniyor;
  public key tauri.conf.json'a manuel kopyalanır. Bu ham deploy adımı
  kullanıcının manual sorumluluğu.
* **Endpoint GitHub Releases** — `updates.showme.app` kendi domain
  yerine direkt GitHub kullandık; ücretsiz, CDN'li, OSS workflow'una
  uyumlu. Round 30+ kendi update host'una geçilebilir.
* **`dialog: true`** — Tauri'nin native diyaloğu çıksın; ancak `apply_update`
  manuel butonla tetiklensin (user opt-in). UX double-confirmation
  güvenli tarafta.
* **`AboutSection` ayrı komponent** — inline JSX 30+ satıra ulaşmıştı,
  ayrıştırma okunabilirlik için.
* **CI workflow yeni dosya, mevcut `build-mac.yml`'i değiştirmedim** —
  tag-push tetiği iki workflow'da aynı; release.yml signed bundle,
  build-mac.yml unsigned smoke. İkisi paralel çalışır, ihtiyaç hâlinde
  birleştirilir.
* **Sidecar build adımı `|| echo`** — ShowMe checkout'u CI'da
  bulunmuyor, sidecar build başarısız olursa workflow durmasın diye
  Round 17'den miras kalan örüntü korundu.
* **`build_release_manifest.py` Python 3.11+** — `dict[str, ...]`
  syntax, dataclass, pathlib; CI Python 3.11 zaten mevcut.
