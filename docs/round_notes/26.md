# Round 26 — LLM planner (Anthropic Haiku + OpenAI fallback) + cost ledger

**Date:** 2026-05-01
**Goal (direktif §0):** Doğal dil → Plan dispatcher LLM-augmented hale
gelsin: Anthropic Haiku 4.5 ana, GPT-4o-mini fallback. Günlük cost cap
varsayılan $1.00 (env ile override). Hata veya cap durumunda Round 19
deterministic planner devreye girer; trader hiçbir zaman karşısında
hata mesajı görmez.

## Delivered

### `src-py/showme/llm.py` (yeni modül)

* **Pricing table** — Haiku 4.5 ($1/M in, $5/M out), GPT-4o-mini
  ($0.15/M in, $0.60/M out). Bilinmeyen model → muhafazakâr yüksek
  estimate (kasıtlı).
* **`CostLedger`** — JSON disk dosyası
  (`~/Library/Application Support/showMe/data/llm_costs.json`); load /
  save / append / today_spend; corrupt JSON → boş ledger ile devam
  eder (planner crash etmez).
* **`Provider`** — `dataclass(name, model, call: ProviderCall)` enjekte
  edilebilir; testler stub'ları doğrudan veriyor.
* **`llm_plan_for(query, *, providers, ledger, function_codes)`**:
  * Önce cap kontrolü; aşıldıysa `CostCapExceeded`.
  * Provider listesinde sıra ile dener, ilk başarılı yanıt:
    `parse_plan_response()` markdown fence'i temizler, JSON parse eder,
    zorunlu key'leri (`intent / action / agents / args`) doğrular.
  * Token sayısı + pricing → `cost_for()` → `CostEntry` ledger'a yazılır.
  * Hepsi başarısız → `LlmPlannerError`.
* **`plan_for_smart(query, *, providers, ledger, function_codes)`**:
  * `providers` boşsa veya LLM hata verirse `plan_for(...)` çağırır.
  * Cap aşıldıysa da deterministic fallback.
  * Genel exception bile yakalanıyor (planner pipe break etmez).
* **`build_default_providers()`** — env'e bakarak Haiku +/veya
  GPT-4o-mini provider'ı kurar; key yoksa boş liste.
* **HTTP transport** — `_anthropic_call` (Messages API) ve
  `_openai_call` (Chat Completions, JSON object response_format=ile).
  `httpx` lazy import (test bağımlılığı yok).

### Orchestrator entegrasyonu

`agents/orchestrator.py` — `ask()` Phase 1 artık providers boş değilse
`plan_for_smart` üzerinden geçiyor; aksi halde önceki `plan_for` ile
aynı.

### `/api/llm/cost` endpoint

`server.py` — GET endpoint cost ledger'ı + cap'i + provider listesini
döndürür. Frontend live cost pill'i için.

### Frontend — Preferences › LLM section

`panes/Preferences.tsx` — yeni `"llm"` bölümü; tabs sıralamasında
`migration` ile `about` arasında. `LlmSection` component'i:

* `today $`, `remaining`, `cap`, `entries`, `state` (open/capped),
  `providers count` SummaryStat grid'i.
* Configured providers Pill chip'leri (boşsa "deterministic only"
  yazısı).
* "Recent calls (last 50)" mini-log: ts (UTC), provider · model,
  in↓ tokens, out↑ tokens, $ cost (>$0.01 → warn renk).
* `⟳` butonu cost endpoint'ini yeniden çeker.

## Decisions / deviations

* **LLM opsiyonel, deterministic her zaman var.** Trader app'i
  çalıştırırken API key'i yok, planner sessizce regex tabanına düşüyor;
  `/api/ask` cevabı her zaman geliyor.
* **Cap default $1.** Direktif §0'da yazan değer; env override
  (`SHOWME_LLM_DAILY_USD`) ile sıfırlanabilir.
* **Pricing local table, dinamik fetch yok.** Anthropic / OpenAI
  fiyat değişikliği yılda bir kez; dinamik fetch sürekli istek
  yaratırdı. Round 28 auto-update yayınlarında pricing tablosu
  güncellenebilir.
* **Provider sırası fallback ile sade.** Birinci başarılı yanıt durur;
  parse hatası bile bir sonraki provider'a geçirir. Bu sayede sözdizim
  hatası veren sağlayıcı konfigürasyonları otomatik atlanır.
* **`response_format: json_object` only-OpenAI.** Anthropic henüz
  zorunlu JSON mod yok; system prompt "Return ONLY the JSON object"
  diyor + `parse_plan_response` markdown fence'i temizliyor.
* **`httpx` lazy import.** `import httpx` provider call gövdesinde
  yapılıyor; CI'da httpx kurulu olmadan testler geçer (test stub'ları
  HTTP'e hiç dokunmuyor).
* **Cost panel Preferences'ta, dedicated pane değil.** N=19 zaten direktif
  hedefinin üstünde; ham ürün pricelı bir gözlem yüzeyini Preferences
  içinde tutmak yeterli. Gerekirse Round 30+ "ASK Cost" pane'i açılır.
