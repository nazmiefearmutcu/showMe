# Round 21 — Real Touch ID prompt + gateLiveTrade + ASK fan-out cards

**Date:** 2026-04-30
**Goal (Round 20 closing list):** `block2`-backed `evaluatePolicy:reply:`
for the real Touch ID prompt + a `gateLiveTrade()` helper, ASK pane
rendering fan-out branches as a 3-card layout, Linux/Windows secrets
fallback messaging.

## Delivered

### Rust shell — real Touch ID prompt

* `Cargo.toml` — added `block2 = "0.5"` to the macOS deps so we can
  manufacture an Objective-C completion block from a Rust closure.
* `biometric.rs` — `run_evaluate_policy(reason, policy)` now creates an
  `LAContext`, wraps a `mpsc::Sender<bool>` in a `RcBlock` closure, and
  invokes `[ctx evaluatePolicy:policy localizedReason:reply:]`. The
  Rust side blocks on `recv_timeout(120s)` waiting for the user's
  decision. The OS modal is a real Touch ID / Face ID prompt now.
* Outcomes:
  * **macOS, biometry available:** `evaluate(reason)` runs the prompt;
    `via: TouchId / FaceId, allowed: <user decision>`.
  * **macOS, only passcode policy:** `via: Password, allowed: <user>`.
  * **macOS, neither policy:** short-circuits to `Denied` with no prompt.
  * **Other targets:** `Ok(false)` so live-trade gates fail closed.

### Frontend — `gateLiveTrade()`

`lib/biometric.ts` gained:

```ts
gateLiveTrade<T>(opts: LiveTradeOptions, action: () => T | Promise<T>): Promise<T>
```

* Defaults to a $1 000 threshold (Rapor 2 §6.7 madde 2). Orders below
  that bypass the prompt; orders ≥ threshold (or with no notional) gate
  on `requireAuth(reason, action)`.
* The 5-min reauth cache from Round 20 still applies — a single
  approval covers the obvious "queue a few orders" flow without
  re-prompting on every click.
* Reason string defaults to `"Live trade · $<notional>"` for OS dialog
  context; callers can override.

### ASK pane — fan-out 3-card layout

When `result.search.branches` is present (briefing intent), the pane
renders a new "Briefing branches" card holding three side-by-side mini
cards — Portfolio / Scan / News — each with:

* Header pill summarizing the leg (`5 pos`, `3L / 2S`, `12 headlines`).
* One-sentence description.
* Up to 5 bullet items (top scan tickers, top headline titles).
* "Open PORT/SCAN/TOP" button that pushes the suggested pane into the
  focused leaf via the same `setFocusedTarget` path used elsewhere.

`describeBranch(key, branch)` does the data-shape probing — handles
both `{data: {data: …}}` (function payloads) and `{data: …}` (direct
list) variants the search agent emits.

### Cross-platform secrets messaging

* `lib/secrets.ts` — `secretsBackend()` returns `"keychain" |
  "browser" | "unsupported"`, picked from `tauri-plugin-os.platform()`.
* Preferences "Secrets" tab now shows a backend pill (Keychain / Browser
  / Unsupported) next to the biometry pill, plus a per-state note:
  * **Browser:** "Keychain only available inside the Tauri shell."
  * **Unsupported:** "This OS doesn't expose the macOS Keychain.
    Round 22+ adds a fallback secret store; for now use environment
    variables."
* The Save button gates on `writable = backend === "keychain"`, so
  Linux/Windows shells ship the surface but lock writes.

## Decisions / deviations

* **Block runs on the calling thread.** `LAContext.evaluatePolicy`
  schedules its block on the main run loop; from a Tauri command
  (which already runs on a tokio task) we just block on the channel
  recv. The OS modal is non-blocking in the user sense — they see the
  Touch ID dialog immediately, the *Rust* call returns when they tap.
* **120-second recv timeout.** Apple's own evaluation timeout is ~60 s;
  we double it to absorb slow networks (Apple Watch unlock confirms
  bouncing through Bluetooth) without leaking a thread forever.
* **`gateLiveTrade` keeps the threshold default at $1 000.**
  Configurable via the option, but the default mirrors the live-trade
  reauth threshold the plan specified rather than picking a new number.
* **3-card grid is fixed-order.** Portfolio / Scan / News reads
  left-to-right the way the briefing flows ("here's what you own,
  here's what's moving, here's why"). Round 22 might let the user
  reorder them via a Preferences toggle.
* **`describeBranch` probes shapes defensively.** ShowMe functions
  return mixed payload shapes (some plain dicts, some `to_dict()` of
  `FunctionResult`). The probe drills one level deep before declaring
  "opaque" — keeps the card useful even if a future search adapter
  changes its envelope.
* **Cross-platform secrets stays advisory.** A real fallback (DPAPI on
  Windows, libsecret on Linux) is a separate dependency vendor each;
  Round 21 ships honest messaging instead. The `secretsBackend()`
  hook is the seam Round 22 will plug.
