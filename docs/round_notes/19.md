# Round 19 — ASKB orchestrator + ASK pane + SCAN polish

**Date:** 2026-04-30
**Goal (Rapor 2 §8 row 19 + Round 18 closing list):** ASKB orchestrator
(Planner / Search / Summarizer / Viz), ASK pane, SCAN `change_pct_today`
sortable column, SCAN drawer keyboard nav.

## Delivered

### Sidecar — `showme/agents/` package

Round 19 adds five thin modules wired together by an orchestrator entry
point. All deterministic; LLM augmentation is reserved for a later round
without changing the contract.

```
showme/agents/
├── __init__.py        re-exports {AskRequest, AskResponse, ask, plan_for, ...}
├── orchestrator.py    ask(req, deps) — pipeline with per-phase timings
├── planner.py         intent classifier + arg extractor
├── search.py          dispatches to scanner / FunctionRegistry, JSON-coerces
├── summarizer.py      narrative + highlight chips
└── viz.py             pane-hint picker (table / chart / cards / metric / split)
```

### Planner — naive intent classifier

Rule-based pattern matching against six intent classes:

| Intent              | Trigger phrases                                        |
| ------------------- | ------------------------------------------------------ |
| `scan`              | scan / find me / hunt / opportunity / overextend …     |
| `portfolio_overview`| portfolio / positions / book / exposure / holdings …   |
| `news`              | news / headline / sentiment / breaking …               |
| `compare`           | compare / vs / versus / difference between …           |
| `lookup`            | what is / who is / describe / info on / tell me about  |
| `function`          | run / open / show me <CODE>                            |

Plus structured arg extractors:

* `_asset_class_hint` — six asset classes pulled from keywords.
* `_extract_symbols` — uppercase tokens, skipping function-code words.
* `_extract_function_code` — only matches against the live FunctionRegistry.
* `_direction_hint` — long / short / bullish / bearish bias for scans.

Two safety nets keep the pipeline useful even on weird queries:

* When `intent == unknown` *and* a symbol was extracted, the planner
  promotes it to `lookup` so a stray "TSLA?" still opens DES.
* When `intent == lookup` but the user typed a known function code, we
  promote it to `function`.

### Search Agent

Routes plans to either the Scanner Agent (Round 17/18) or the ShowMe
`FunctionRegistry`. New `_jsonify(value)` helper recursively coerces
pandas Series / DataFrames, numpy scalars, datetimes, and the
`NaN`/`Inf` floats that FastAPI's pydantic 2 serializer otherwise
chokes on. Verified by feeding FA on MSFT through `/api/ask`: 4 081 ms,
≈ 200 fundamental keys, no 500.

### Summarizer Agent

Deterministic narrative builder per intent:

* `scan` → `"Scanner produced N candidates from <universe>. M LONG, K SHORT.
  Currently held: AAPL, MSFT. Overextended: NVDA. Top: BTC (L 80), ETH (L 70)…"`
* `function` → `"Ran FA on MSFT. Took 4080 ms. Sources: sec_edgar.
  200 record(s) returned." / "Top keys: …"`
* `compare` → suggests an explicit "Open DES + GP" layout when symbols are
  parsed.
* fallback → educates the user about supported intents.

Each narrative ships with a `highlights[]` array of `{label, value, tone}`
that the ASK pane renders as Pill chips.

### Viz Agent

Picks a pane hint from the search result shape:

| Search kind                        | Viz pick |
| ---------------------------------- | -------- |
| `scan`                             | `table` + open SCAN hint |
| `function` `code in {GP, TECH}`    | `chart` + GP/<sym> hint |
| `function` `code == PORT`          | `table` + PORT hint |
| `function` `code in {DES, FA}`     | `cards` + DES/FA <sym> hint |
| `compare`                          | `split` + two DES panes |
| `noop`                             | `none` |

The frontend uses these hints to render an "Open DES · AAPL" button that
pushes the suggested pane into the focused leaf via `setFocusedTarget`.

### Sidecar `/api/ask`

```http
POST /api/ask
{ "query": "find me crypto opportunities high conviction" }
```

Returns the full `AskResponse` envelope: `{plan, search, narrative,
highlights, viz, phases, elapsed_ms, warnings}`. The four phases — plan,
search, summarize, viz — each report their own `elapsed_ms` so the UI
can show timing pills the same way SCAN does.

### Frontend — ASK pane

`functions/ASK.tsx`:

* Textarea + 5 sample-query chips (`⌘↵` runs).
* Result block in a 2-column grid: narrative card (with phase pills +
  highlight chips) and a plan card (intent / action / agents / rationale).
* "Suggested view" card with a Pill row + Open buttons for every pane
  hint the Viz Agent produced. Clicking pushes the suggested
  `(code, symbol)` into the focused leaf via `setFocusedTarget` and
  navigates to the matching hash route.

Registered as `ASK` in the pane registry.

### SCAN polish

* New `Δ today` sortable column rendering `fine.quote.change_pct` with
  a signed `ChangeText`. Three sortable headers — Conf %, Score,
  Δ today — toggle `sortKey` state; sorting always uses `|score|` /
  `|change_pct|` so LONG and SHORT picks both bubble up.
* Drawer keyboard nav: `←` / `→` move between rows in current sort
  order, `⌘↵` jumps to DES on the focused symbol, `Esc` closes the
  drawer.
* Drawer renders a hint footer `"← / → between rows · ⌘↵ Open DES · esc close"`
  when there's more than one row.

## Decisions / deviations

* **No LLM in Round 19.** Rapor 2 §8 row 19 says "Bilinen sorgu setinde
  reasonable sonuç" — the deterministic planner is sufficient for the
  named query set. The `Plan` shape is the same in either world; an
  LLM augmentation slots in at the orchestrator's planner step.
* **Search Agent owns serialization.** `_jsonify` lives in `search.py`
  rather than a generic util because the only caller right now is the
  function-result coercion path. When orchestrator response inflation
  starts hitting other shapes we'll promote it to a `serialize.py`.
* **Phase pills reuse the SCAN convention.** Same look, same Pill
  primitive — keeps the UI mental model "phases are little pills with
  ms timings" consistent across the app.
* **Drawer keyboard nav is global, not pane-scoped.** Inside SCAN the
  drawer is the only "selected row" surface, so listening on `window`
  for arrow keys is fine. When a second pane gains a similar drawer
  we'll factor a shared `useDrawerNav(rows, current)` hook.
* **Sort by absolute value.** Both `score` and `change_pct` sort by
  magnitude so a SHORT idea with `score = −0.95` ranks alongside a LONG
  with `score = +0.95`. The signed value still drives the row colors.
* **Function intents propagate symbols.** Round 18's "FA on MSFT" was
  failing because the function intent didn't pass an instrument; now
  the search agent reads `args.symbols[0]` and binds it.
