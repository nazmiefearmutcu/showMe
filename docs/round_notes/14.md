## Delivered

### Sidecar: `/api/fn/{code}`

Round-12's `/api/proxy/{path}` was a 501 stub. Round 14 ships a proper
typed gateway that:

* triggers `function_factory.get_factory()` so registry decorators fire
  (idempotent — `_factory` singleton on subsequent calls);
* resolves the function class via `FunctionRegistry.get(code.upper())`;
* binds `symbol`/`asset_class` query params into a fresh `Instrument`
  (when present);
* runs `await fn.execute_timed(instrument=…, **params)`;
* returns the `FunctionResult.to_dict()` directly.

Both `GET` (query params) and `POST` (JSON body) are supported. The legacy
`/api/proxy/*` now returns **410 Gone** with a clear deprecation note.

### Frontend client (`lib/functions.ts`)

* `runFunction<T>(code, { symbol, asset_class, params, signal })` — picks
  GET/POST based on whether any param value is a non-primitive.
* `FunctionCallError` — typed error with status + body.
* `useFunction<T>` hook — minimal `idle | loading | ok | error` state,
  AbortController on unmount + dep change, params stringified for stable
  dep tracking.

## Decisions / deviations

* **Pane registry is plain object, not a hook-based provider.** Adding the
  next 5 panes is a single line — `DES: DESPane,` etc. — no boilerplate.
* **TECH is an alias for GP today.** ShowMe has both functions; the chart
  surface is identical for now. When TECH grows its own indicator-tuning
  UI it splits into a dedicated `TECHPane`.
* **`useFunction` does not cache across panes.** That's fine for Round 14
  — each pane mount triggers one fetch, and re-mounts during nav are
  cheap enough in browser-mode. Round 15 introduces a SWR-style cache
  if/when GoldenLayout pane swaps become frequent.
* **Lightweight-charts v4 (not v5).** v5's new generic `addSeries(SeriesDefinition,…)`
  required `addSeries(CandlestickSeries, …)` which ShowMe's chart returns
  cleaner with v4 imperative API. We can lift to v5 in Round 15 when the
  ecosystem catches up.
* **POST vs GET decision is automatic.** `runFunction` walks the param
  values once: any object/array → POST. Keeps the hot path lean while
  letting `STRS{symbol_shocks: {BTCUSDT: -0.5}}` work without callers
  thinking about transport.
* **EQS uses local state**, not `useFunction`, because the user explicitly
  presses "Run". Auto-fetch would burn yfinance budget every keystroke.
