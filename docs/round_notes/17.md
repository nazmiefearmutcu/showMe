# Round 17 — Scanner Agent + per-window workspace + link groups

**Date:** 2026-04-30
**Goal (Rapor 2 §8 row 17 + Round 16 closing list):** Scanner Agent (Phases
A+B), per-window workspace state, cross-pane symbol broadcast.

## Delivered

### `showme/scanner.py` — Phase A + Phase B

* `ZAK_MATRIX` per asset class (EQUITY/ETF/CRYPTO/FX/COMMODITY/BOND/MACRO).
  Crypto numbers preserved verbatim from ShowMe §bot_service._ZAK; multi-asset
  numbers are calibrated so longer TFs dominate equity/macro and shorter
  TFs matter more for FX.
* `UNIVERSES` — 6 static universes (EQUITY:US:LARGE, EQUITY:US:SP500_TOP30,
  CRYPTO:MAJORS, FX:G10, COMMODITY:CORE, ETF:US:CORE).
* `consensus_signal(closes)` — pure-Python RSI(14) + MACD(12,26,9) +
  50/200-MA cross with documented mean-reversion bias on the RSI piece.
  No NumPy / SciPy / TA-lib.
* `select_universe(req)` — Phase A naive intent router (crypto / fx /
  commodity / etf / equity heuristics). Round 19 swaps in the LLM
  Planner.
* `run_scan(req, deps)` — Phase B coarse scan. For each symbol × top-3
  ZAK timeframes, `_fetch_closes()` calls the right ShowMe adapter
  (`ccxt_failover` / `coingecko` for crypto, `yfinance` for everything
  else). Each timeframe contributes `sign × confidence × ZAK_weight` to
  the symbol's aggregate score. Concurrency capped at 8 (asyncio.Semaphore).
  Output is a `ScannerResult` envelope with phase timings + warnings.

### Sidecar endpoints

* `GET /api/scanner/universes` — list available universes.
* `POST /api/scanner/run` — body `{intent, universe?, asset_class?,
  timeframes?, top_n?}` → `ScannerResult.to_dict()`.

Both endpoints reject when ShowMe isn't attached (503).

### `SCAN` pane

`functions/SCAN.tsx` — intent textarea + sample chips + universe override
select + Top-N + Run button. Result block renders phase pills (A.universe,
B.coarse_scan elapsed), result count, and a clickable DataGrid.

Clicking a row pushes `(DES, symbol)` into the focused leaf via the same
`setFocusedTarget` path used by ⌘K — so the user can run a scan in one
pane and jump to DES in a sibling (with link group set, sibling rebinds
automatically).

  `[A-Za-z0-9_-]` to `_`. State files become

## Decisions / deviations

* **Static universes in Round 17.** Phase A's "intent → universe"
  routing is a 4-keyword heuristic. Round 19 swaps in the LLM Planner;
  the contract (`ScanRequest` shape, `select_universe` return tuple)
  stays stable so the call sites don't move.
* **Concurrency cap = 8.** yfinance circuit-breaker tolerates ~10
  parallel requests on a fresh boot; 8 leaves headroom for the
  function-index sidecar might still be loading.
* **No `bot_service.ScannerService` import.** That class owns live bot
  state (cycle count, lock files, …) and would block on the live trade
  loop. We ship a fresh service that only *re-uses* the ZAK matrix
  concept + adapter chain.
* **RSI saturation bias kept.** Pure deterministic uptrends max out RSI
  → -1 (overbought) score, which combined with +0.8 MACD + +0.6 MA
  produces a NEUTRAL aggregate. This is the documented mean-reversion
  bias; a "trend-only" mode arrives in Round 19's tuner UI.
* **Link groups: pane-level, not workspace-level.** A trader can have
  panes A/B linked on AAPL and panes C/D linked on MSFT in the same
  window. Symbol broadcast scoped to the explicit group keeps the
  mental model simple.
* **Per-window state, with migration.** When users upgrade from Round
  16's single `workspace.json`, the first load on the `main` window
  finds the legacy file and replays it. Subsequent saves write
  `workspace-main.json`; the legacy file becomes inert.
* **DataGrid row click is opt-in.** Existing PORT/EQS callers don't
  pass `onRowClick`, so they keep their per-cell anchors (PORT's
  symbol → DES button, EQS's symbol → DES button). New code prefers
  the typed `onRowClick`.
