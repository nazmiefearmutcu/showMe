Spec:  [docs/superpowers/specs/2026-05-21-multi-exchange-portfolio-foundation-design.md](../docs/superpowers/specs/2026-05-21-multi-exchange-portfolio-foundation-design.md)

## What landed

* Credential vault (`brokers/credential_store.py`) backed by macOS Keychain (service `com.showme.exchanges`) with in-memory backend for tests via `SHOWME_CREDENTIAL_BACKEND=memory`.
* `CcxtBroker(BaseBroker)` adapter wrapping ccxt-async — one adapter class covers ~111 crypto exchanges from a single registry.
* Exchange catalog (`brokers/catalog/exchanges.yml`) — 112 entries (111 ccxt + 1 alpaca-live), regenerated by `scripts/build_exchange_catalog.py` with CI drift check.
* Per-credential dynamic broker registration in the factory (`brokers/factory.py`) — `register_credential`, `replay_stored_credentials`, async `close_all_brokers` with `_LIVE` tracking.
* `/api/exchange/*` routes: GET catalog / GET credentials / POST credentials / DELETE credentials/{id} / PATCH credentials/{id} / POST credentials/{id}/test.
* Sidecar boot replay so saved credentials survive restart.
* UI: `exchange-store` zustand store + `CONN` pane (search + filter chips + dynamic form + Test/Delete/Upgrade) + sidebar Connections group.

## Frozen contracts

* Keychain service name: `com.showme.exchanges`
* Broker name format: `{exchange_id}:{credential_id}`
* Permission tuples: `("read",)` / `("read", "trade")`
* Credential index file: `$SHOWME_HOME/credentials.json`
* `SHOWME_CREDENTIAL_BACKEND=memory` selects the in-memory backend
* `CredentialPatch` accepts `permissions` + `confirm_account_label` only (no rename via PATCH)
